3. 连接建议

如果连接卡在"连接中"，可能的原因：

  • 服务器配置不正确（IP、端口、用户名、密码）
  • 目标服务器不可达
  • SSH 服务未运行在目标服务器上
  • 网络防火墙阻止了连接
  • 认证失败（错误的用户名/密码）

临时解决方案：

在配置文件中添加一个可用的服务器配置：

{
  "host": "your-server-ip",
  "port": 22,
  "username": "your-username", 
  "authType": "password",
  "password": "your-password"
}
